Exercises: Variable Scopes in JavaScript(1)

  1. outer; inner; outer
    • The console.log(a) sees the global scoped a assigned to "outer"
    • testScope()'s scope sees the locally declared a which is assigned to "inner"; this function logs this value of a
  2. outer; inner; inner
    • The variable a is the global variable both when it is assigned to "outer" in line 1 as well as line 4 within the function testScope
    • Thus, the first console.log(a) yields "outer"
    • testScope() is invoked, which reassigns the global variable to "inner" and then logs that value
    • Since 'a' was reassigned from the function invocation, line 10 logs "inner" to the console
  3. basket; play station; computer
    • the global variable basket is first assigned to "empty"
    • line 26 invokes the goShopping function
    • the three local functions are hoisted to the top of this function
    • console.log(basket); yields the global basket variable which is assigned to "empty"
    • shop1 function is invoked setting global variable basket to "tv"
    • shop2 function is invoked setting global variable basket to "computer"
    • shop3 function is invoked, it shadows the variable basket, creating its own local variable basket and assigns this variable to "play station" and then logs this value
    • goShopping ends with a log to the global variable basket which was last assigned to "computer"
  4. undefined
    • variable declaration of a is hoisted above console.log(a), however variable assignment occurs after console.log(a)
    • Thus, a is undefined when console.log(a) is called
  5. Uncaught ReferenceError: a is not defined
    • hello is hoisted to the top but is never invoked to declare or assign the global variable a.
  6. hello
      hello() is invoked in this case, declaring 'a' and assigning "hello" to it before line 6
  7. Uncaught ReferenceError: a is not defined
    • the variable in hello() is now a local variable to this function and does not declare the global variable nor sets it to any value
